ワークショップの実施にあたって、いくつかの注意点や前提条件を共有します。

## ゴール

以下を目指します。

- セキュリティへの関心向上
- パスキー認証の基本的な概念、仕組みについて理解を深める
- サーバサイドの実装例を学ぶ (Flask)
- ~~クラウドサービス (Vercel) へのデプロイを行い、動作確認する~~

## やらないこと

- クライアントサイドの詳しい実装、解説
  - 特にスマートフォン含むネイティブ実装は取り扱いません。
- 商用レベルのセキュアな実装やプラクティスの紹介
  - 紹介するコードは、技術的な教育を目的としたもので、業務や商用環境での利用を意図しておりません。
- クラウドサービスの細かな設定方法や使い方

## 参加者に期待するスキル

- 基本的なプログラミング、特に Python の文法に関する知識
- Web 技術の基礎知識
  - HTML、JavaScript
  - HTTP、クライアント/サーバモデル
  - 公開鍵暗号方式

これらのスキルがあれば、十分に理解できる内容を想定しています。

## 作業環境

任意の開発環境で問題ありませんが、推奨環境を紹介します。

- Python 3.9 以降が利用できること
- エディタとして VSCode が利用できること
  - 他のエディタでも問題ありませんが、コードの処理を追うためにデバッガを利用します
- ブラウザとして Google Chrome が利用できること
  - 他の Chrome ベースのブラウザ (Edge、Safari)でも問題ありませんが、サポートは Chrome のみを対象とします
